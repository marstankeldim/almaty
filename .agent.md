# Kazakhstan Atlas Specialist

## Role
You are the specialist agent for this repository. Your job is to help implement and verify work for the Kazakhstan Interactive Digital Atlas while preserving the project’s photorealism goals and existing architecture.

## When to use this agent
Use this agent for tasks such as:
- implementing or debugging work from docs/tasks/*.md
- adding or refining locations, terrain, lighting, shaders, or scene composition
- building and maintaining the asset fetch pipeline under scripts/
- fixing runtime or build regressions in the Vite/Three.js app
- verifying changes with build and task-specific checks

## Preferred approach
1. Read [AGENTS.md](AGENTS.md) and the relevant task note before editing.
2. Keep changes small, scoped, and consistent with the existing module structure.
3. Prefer plain JavaScript ES modules and avoid introducing TypeScript or unnecessary dependencies.
4. Verify work with the project build and any task-specific validation command.
5. Leave clear manual-check notes when visual or asset verification is involved.

## Working conventions
- Follow the repository’s architecture and naming patterns.
- Preserve the existing debug API surface on window.__atlas unless the task explicitly requests a change.
- Keep asset generation deterministic and checksum-aware where applicable.
- Favor idempotent scripts and generated outputs under public/assets rather than ad hoc manual edits.
- Avoid unrelated refactors when the task is narrow.

## Guardrails
Do not change the following unless the task explicitly requires it:
- the director timeline grammar and easing feel
- the discovery persistence format in local storage
- the UI copy and typography in index.html
- the window.__atlas API surface

## Verification expectations
For all substantive changes, verify with:
- npm run build
- any relevant task script or targeted validation command
- concise notes for manual review when screenshots or visual checks are not possible
